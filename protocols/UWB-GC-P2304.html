GC-P2304-GS-2 UWB 串口协议冻结文件

当前已知

  • 模块：GC-P2304-GS-2。
  • 接口：UART，不是 SPI。
  • 供电：当前样机按 3.3V。
  • ESP32 引脚：TX GPIO17 -> UWB RX；RX GPIO18 <- UWB TX。
  • 默认波特率候选：115200，最终以卖家资料/实测为准。

禁止猜测

目前项目内没有可靠的 GC-P2304-GS-2 串口帧头、长度、校验、字段定义。AI 不得编造帧格式。

正式实现 uwb_gc_p2304.cpp 前，必须补齐以下任一证据：

  1. 卖家/厂家协议 PDF 或截图；
  2. 串口抓包样例 20 行以上，覆盖距离/角度变化；
  3. 原厂 demo 代码。

目标解析字段

最终驱动必须输出：

struct UwbTarget {
  bool valid;
  uint32_t last_update_ms;
  int distance_mm;
  float bearing_deg;
  uint8_t confidence;
};

临时实现规则

在协议未冻结前，只允许写：

  • UART 初始化；
  • 原始串口日志记录（raw log）；
  • 超时检测；
  • mock parser 单元测试；

禁止写“看起来像”的帧解析逻辑；不得实现猜测性 parser，更禁止让 AUTO_FOLLOW 依赖未验证字段。

布局与 EMI 要求

UWB 模块/天线应远离 DC-DC Buck、电池主线、控制器相线和大电流端子；与 DC-DC 模块/电感至少保持 50mm。DC-DC 5V 输出端建议并联低 ESR 电解电容（470uF 起步，耐压 ≥10V）和 0.1uF 陶瓷电容。若测距出现跳变/丢包，优先检查 Buck 纹波、天线附近金属/电源线、模块供电稳定性。
