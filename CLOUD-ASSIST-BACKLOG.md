# FollowBox 云端困难场景协助待办

> 阶段定位：等本地基础能力完成后再研发。基础包括：手动遥控、安全链、急停反馈、低速运动、UWB 跟随、TOF/超声本地避障、H5 telemetry、日志闭环。

## 目标

当 FollowBox 在复杂场景中 `STUCK`（不知道怎么走、前方障碍、目标被遮挡、局部绕障失败）时，车端通过 WiFi 上传结构化状态和低帧率图像，由云端 AI/规则引擎给出低速、短距离、可验证的动作计划；ESP32-S3 本地 `command_guard` 根据急停、遥控器、传感器和模式权限做最终安全裁决。

## 核心原则

1. 云端不直接输出 PWM，不做高速实时闭环。
2. 云端只输出动作原语：`STOP`、`HOLD`、`TURN_LEFT/RIGHT`、`NUDGE_FORWARD/BACKWARD`、`AVOID_LEFT/RIGHT`、`RETRY_FOLLOW`、`ASK_HUMAN`。
3. 单次动作必须短、慢、可中断，例如 10-30cm / 10-30° / 1-2s 内过期。
4. ESP32-S3 本地拥有最终否决权：急停、遥控器失联、障碍过近、传感器未知、命令过期都拒绝执行。
5. 第一版先“只建议不执行”，第二版“人确认后执行”，第三版才允许极低风险自动执行。

## 待办路线

### MVP1：云端只建议

- [ ] 定义 `CLOUD-ASSIST-SPEC.md`。
- [ ] 定义 `telemetry` JSON schema。
- [ ] 定义 `stuck_event` JSON schema。
- [ ] 定义 `cloud_command` JSON schema。
- [ ] H5 显示云端建议，不自动执行。

### MVP2：人确认执行

- [ ] H5 增加“允许执行本步动作”按钮。
- [ ] ESP32 增加 `REMOTE_ASSIST` 模式。
- [ ] ESP32 增加 `command_guard`：命令过期、限速、限距、障碍物、急停、遥控器在线、模式权限检查。
- [ ] 每次只执行一步，执行后重新上传状态。

### MVP3：低风险自动执行

- [ ] 仅允许 `STOP`、`HOLD`、`RETRY_FOLLOW`、低速小角度转向、小距离后退自动执行。
- [ ] 连续失败 3 次自动 `ASK_HUMAN`。
- [ ] 所有云端动作写日志：请求、建议、裁决、执行结果、拒绝原因。

### MVP4：云端协助绕障

- [ ] 接入 ESP32-S3-CAM 低帧率 JPEG，优先“卡住时拍照上传”，不要先做连续视频。
- [ ] 云端规则引擎先判断明显场景：左/右空间、前方阻挡、UWB 目标方向。
- [ ] 视觉模型只做辅助：判断通道/箱子/人/墙/不可通行区域。
- [ ] 大模型只做综合推理和动作计划生成，输出结构化 JSON。

## 后续研究方向

- 边缘-云协同：本地安全/短反射，云端慢推理。
- Shared autonomy：人/AI 给高层意图，本地控制器保证可行和安全。
- Confidence-aware planning：云端输出置信度，低置信度只建议或请求人工。
- VLM grounding：用视觉语言模型识别“左侧可通行/前方箱子/目标被遮挡”。
- 小模型边缘部署：未来可在上位机/Raspberry Pi/Jetson 上跑小 VLM/SLM，ESP32 仍只做实时控制。

## 禁止事项

- [ ] 禁止云端直接控制 PWM。
- [ ] 禁止云端绕过急停/遥控器/本地避障。
- [ ] 禁止 WiFi 断开时继续自动动作。
- [ ] 禁止在斜坡、道路、人群、儿童/宠物附近做早期自动测试。
- [ ] 禁止一次下发长距离连续路径并盲跑。
