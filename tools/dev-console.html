部署云端服务
上传 cloud/ 到服务器 → PM2 重启 → Nginx 刷新
将本地 cloud/server.js + cloud/public/ + cloud/package.json 通过 SCP 上传到服务器，然后执行 deploy-clean-cache.sh（PM2 重启 + Nginx reload）。
📦
编译固件并上传到云端
pio run → 生成 manifest → scp 上传 → PM2 重启
编译 ESP32 固件，自动生成 manifest.json（含版本号、MD5、大小），上传 bin + manifest 到云端 /www/wwwroot/followbox-cloud/firmware/。设备轮询时会发现新版本并触发 OTA。
🧪

ESP32 设备状态
检测 192.168.4.1:3232 OTA 端口是否在线
确保电脑已连接 ESP32 的 WiFi AP (FollowBox-xxx)
⚡
WiFi OTA 刷写
pio run → pio run -t upload --environment ota
编译固件后通过 WiFi 直接 OTA 刷写到 ESP32（192.168.4.1:3232）。
⚠️ 需要电脑已连接 ESP32 AP，OTA 过程中设备会重启，约 30-60 秒。
☁️
云端 OTA 管理
查看云端固件版本 + 触发强制 OTA
云端 OTA 流程：先在「云端部署」面板编译并上传固件 → 然后点「触发强制 OTA」将 manifest 的 force=true，ESP32 下次轮询时立即下载新固件并刷写。
