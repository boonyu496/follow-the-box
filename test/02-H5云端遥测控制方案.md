## 3. 已有的基础设施（已实现代码）

### 3.1 云端服务器 (`cloud/server.js`)

已实现的 API：

| 端点 | 方法 | 功能 | 安全 |
|------|------|------|------|
| `/api/device/{id}/ingest` | POST | 设备上传遥测+日志 | token 认证 |
| `/api/device/{id}/command` | GET | 设备轮询待执行命令 | token 认证 |
| `/api/device/{id}/command` | POST | 操作员下发命令 | Bearer token |
| `/api/device/{id}/events` | GET | SSE 实时推送 | — |
| `/*` | GET | 静态 H5 面板 | — |

**关键特性：**
- 设备数据基于内存 Map（无数据库，适合单机调试）
- SSE 实时推送状态给浏览器
- Command TTL 750ms：deadman 超时自动停车
- CORS 全开（调试阶段）
- 日志环回缓冲区最多 500 条

### 3.2 ESP32 云端客户端 (`cloud_client.cpp`)

| 功能 | 实现 |
|------|------|
| 上传间隔 | 1000ms（可配） |
| 命令轮询间隔 | 150ms（可配） |
| 超时停车 | `CLOUD_LOST_STOP_MS` 配置（默认 750ms） |
| Seq 防重放 | 递增 seq，旧 seq 拒绝 |
| Deadman 安全 | deadman=false→停车 |
| 安全停请求 | safe_idle=true→立即停车+锁定 |
| 并发安全 | portMUX 自旋锁保护输入结构体 |
| 日志上传 | debug_console 环形缓冲区 JSON 序列化 |
| 线程安全 | comm_task（Core0）中调用，不阻塞 control_task |

### 3.3 ESP32 端配置 (`cloud_config.h`)

### 4.1 云服务器部署

**推荐方案：宝塔面板部署（与你已有 boonai.cn 一样）**

```text
boonai.cn 服务器

│   ├── firmware/                    ← OTA .bin 文件存放
```

## 5. 安全边界（严格审查）

### 5.1 云端不能做什么（硬限制）

| ❌ 禁止 | 原因 |
|--------|------|
| 云端直接设置 PWM/油门值 | 绕过本地安全链 |
| 云端清除急停状态 | 物理急停必须物理恢复 |
| 云端绕过安装向导 | 未完成安全确认前禁止 AUTO_FOLLOW |
| 云端绕过速度限制 | `MANUAL_CLOUD_LOW_SPEED` 最低速 |
| 云端远程解锁 FAULT_LOCKOUT | 必须本地人工确认安全 |
| 云端自动进入 AUTO_FOLLOW | 必须 UWB 有效且人工确认 |

### 5.2 云端调试安全流程

```
H5 远程点动操作流程：

1. 确认车轮架空 OR 场地空旷、无人员/障碍
2. 确认物理急停在手边
3. 点击 H5 "请求控制"
4. 按住 Deadman 按钮（必须有死人手柄）
5. 松手 → 750ms 内无新命令 → 自动停车
6. 随时可按物理急停
```

## 7. 与现有代码的兼容性

| 现有功能 | 云端方案是否影响 | 说明 |
|---------|:--------------:|------|
| DS600 遥控 | ❌ 不影响 | DS600 优先级高于云端 |
| H5 本地面板 | ❌ 不影响 | 本地 H5 和云端 H5 共存 |
| UWB 跟随 | ❌ 不影响 | AUTO_FOLLOW 模式下云端命令不生效 |
| 安全门控 | ❌ 不影响 | safety_manager 统一裁决所有控制源 |
| 急停按钮 | ❌ 不影响 | 物理急停永远最高优先级 |
| WiFi 状态 | ⚠️ 影响 | 默认 SoftAP 改为 STA 时，本地局域网需通过路由器访问 |

当前 cloud_client 代码已完全实现`CLOUD-TELEMETRY-SPEC.md` 规范，默认 `DISABLED`，启用只需：
1. 改 `cloud_config.h` 中的 `ENABLED` 和 URL/token
2. 改 `network_config.h` 中的 WiFi 凭据
3. 编译 `build_flags` 加 `-D FOLLOWBOX_WIFI_STA=1 -D FOLLOWBOX_CLOUD_ENABLED=1`
4. 首烧到 ESP32
